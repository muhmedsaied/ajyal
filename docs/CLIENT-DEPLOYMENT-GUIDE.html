3. Version Naming Convention

3.1 Semantic Versioning Format

Version Format

v{MAJOR}.{MINOR}.{PATCH}[-{QUALIFIER}]

3.2 Version Components

ComponentDescriptionIncrement When
MAJORMajor versionBreaking changes or major redesign
MINORMinor versionNew features (backwards compatible)
PATCHPatch versionBug fixes (backwards compatible)
QUALIFIEROptional suffixPre-release, build number, hotfix

3.3 Version Examples

VersionTypeDescription
v1.0.0InitialFirst production release
v1.0.1PatchBug fix release
v1.1.0MinorNew feature added
v2.0.0MajorBreaking changes
v1.0.0-rc1Pre-releaseRelease Candidate 1
v1.0.0-hotfix.1HotfixEmergency fix
v1.2.3-build.456BuildWith CI build number

3.4 File Naming

PlatformFormatExamples

4. How to Upload

4.1 Using AWS CLI

Upload Windows Package

# Upload to S3
aws s3 cp your-package.zip \
    s3://preprod-ajyal-deployments-946846709937/windows/app/v1.0.0.zip \
    --profile ajyal-deploy

# Verify upload
aws s3 ls s3://preprod-ajyal-deployments-946846709937/windows/app/ \
    --profile ajyal-deploy

Upload Linux Package

# Upload to S3
aws s3 cp your-package.tar.gz \
    s3://preprod-ajyal-deployments-946846709937/linux/botpress/v1.0.0.tar.gz \
    --profile ajyal-deploy

# Verify upload
aws s3 ls s3://preprod-ajyal-deployments-946846709937/linux/botpress/ \
    --profile ajyal-deploy

4.2 Using AWS Console

  1. Navigate to: https://console.aws.amazon.com/s3
  2. Select bucket: preprod-ajyal-deployments-946846709937
  3. Navigate to the appropriate folder (e.g., windows/app/)
  4. Click Upload
  5. Select your package file
  6. Click Upload
